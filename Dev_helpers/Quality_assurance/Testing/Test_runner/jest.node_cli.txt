Jasmine differences:
Uses Jasmine under the hood:
  - same:
     - [x|f]describe|it|before|AfterEach|All()
     - assertions
  - different:
     - CLI, programmatic usage
     - CONF
     - does not use Jasmine Ajax
  - uses older version 2.5.2-light, i.e.:
     - no EXPECT.nothing|toBePositive|NegativeInfinity|toThrowMatching|toHaveClass()
     - no JASMINE.truthy|falsy|empty|arrayWithExactContents()
  - optional differents names:
     - x|fdescribe|it -> describe|it.skip|only()
     - it[.skip|only] -> test[.skip|only]()
     - JASMINE.anything|any|stringMatching|arrayContaining|objectContaining -> expect.anything|...
  - `this`:
     - top-level `this` is an empty object
     - describe|it() `this` is not shared
     - i.e. must use arrow functions and reuse that `this` object
        - i.e. better to just not use `this` but share a top-level object
  - Jasmine assertions:
     - EXPECT.toBe() uses Object.is() not ===
     - EXPECT.toThrow() cannot take an argument
     - no JASMINE.addCustomEqualityTester()
  - Jasmine timers???
  - Jasmine spies???
