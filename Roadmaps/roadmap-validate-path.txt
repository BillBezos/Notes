validatePath("PATH"[, OPTS])->"PATH":
  - opts (in order):
     - opts.defaultValue VAL
     - URI(file://) or BUFFER -> STR
        - error if not STR
     - opts.child BOOL (def: undefined): must be child of its base
     - opts.base "DIR_PATH" (def: process.cwd()): path.resolve()
        - if relative itself, relative to process.cwd())
        - opts.posix BOOL (def: undefined): call path.posix|win.resolve() instead
     - no null bytes
     - check whether windows char check is necessary https://github.com/antarasi/path-validation/blob/master/README.md
     - strip final backslash
     - filenameFilter|pathFilter REGEXP|FUNC(STR)->BOOL|"ERROR"
     - opts.exists BOOL (def: undefined)
        - if files does not exist, does not do any of the following, except opts.can* using first existing ancestor (make sure it works)
     - opts.dir BOOL (def: undefined)
     - opts.allowSpecial (def: false)
     - opts.canRead|Write|Execute (def: true|undefined|undefined)
        - recursive??? (i.e. can read|write|exec PATH children)
     - statFilter(STAT)->BOOL
     - options.followSymlinks BOOL (def: false)
        - done at end, i.e. not for above

Divide some of the above into smaller utils

options validation:
  - i.e. from an object
  - several in parallel
  - can make the base of one option another option
      - should avoid recursion
